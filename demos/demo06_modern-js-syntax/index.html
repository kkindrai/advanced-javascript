Modern Javascript Syntax

Arrow Syntax

We should be using this new "Arrow Syntax" moving forward

        
            // The Renaissance (The Ideal "Arrow Syntax")
            const theRenaissance = () => {
                "use strict";
                return "The Renaissance";
            }

One Liner Arrow Syntax

Arrow syntax, but the return is #implied. we love conversational cues

  • {}'s are not necessary in this format
    •         
            const functionName = () => "Returned String";

    Syntax With Parameters

    There exist two primary ways to do this...

    Single Parameter Statements

    In single parameter statements, we can omit the ()'s, as they are implied

            
            const functionName = parsedParameter => {
                "use strict";
                console.log(parsedParameter);
                return parsedParameter;
            }

    Multiple Parameter Statements

    In multiple parameter statements, we do as we think we should, and keep the ()'s

            
            const functionName = (param1, param2) => {
                "use strict";
                console.log(param1 + param2);
                return param2;
            }
